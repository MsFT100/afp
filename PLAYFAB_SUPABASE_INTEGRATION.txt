PROJECT OVERVIEW: CRM ARCHITECTURE (PLAYFAB & SUPABASE)

1. THE ROLE OF PLAYFAB (The "Game" Backend)
PlayFab is the source of truth for all end-user (Player) data. Within this project, 
it is used for:

   - Player Identity: Managing Master Player Accounts, Display Names, and 
     Lineage/Entity IDs.
   - Search & Discovery: Using the modern Entity Search API (SearchProfiles) to 
     list players and filter them by segments, emails, or IDs.
   - Economy Management: Adjusting Virtual Currency balances (granting/subtracting) 
     directly from the CRM dashboard for support or promotional reasons.
   - Promotions & Metadata: Storing and retrieving UserData (KVP) such as 
     'promoCode' and 'referralCode' to track marketing campaign effectiveness.
   - Telemetry: Reading Statistics (e.g., scores, levels) to display player 
     progress in the dashboard.
   - Security: Handling the Entity Programming Model through Entity Tokens 
     and Developer Secret Keys for administrative actions.

2. THE ROLE OF SUPABASE (The "Admin" Infrastructure)
While PlayFab manages the players, Supabase manages the dashboard users (Promoters, 
Admins, and Staff). Based on the context:

   - Admin Authentication: Supabase (via GoTrueClient) handles the login, 
     logout, and session management for the people accessing this CRM.
   - Route Protection: The /(protected)/ directory structure suggests 
     Supabase Auth Guards are used to ensure only authorized staff can 
     view player data or modify currencies.
   - CRM Database: While not shown in the snippets, Supabase typically stores 
     CRM-specific data that doesn't belong in a game engine, such as audit logs 
     of which admin changed a player's currency, or internal notes on player 
     support tickets.

3. HOW THEY WORK TOGETHER
The application acts as a bridge between the two:
   - AUTH FLOW: A user logs into the Next.js app via Supabase.
   - API FLOW: Once authenticated, the PlayFabAPI class is initialized with 
     environment variables to perform high-level administrative tasks on 
     the game's player base.
   - UI MAPPING: The Promotions Page fetches a list of players from PlayFab, 
     parses their PlayFab-specific UserData, and presents it in a React 
     interface built for CRM staff.
