# CRM Backend Architecture (MySQL + NestJS)

## 🎯 Goal
Replace:
- PlayFab → Player system + economy + stats
- Supabase → Admin auth + CRM data

With:
- NestJS backend
- MySQL database
- Full ownership of logic

---

## 🧱 Tech Stack

- Backend: NestJS (Node.js + TypeScript)
- Database: MySQL
- ORM: TypeORM
- Auth: JWT
- Cache (optional): Redis
- Realtime (optional): WebSockets

---

## 📂 Project Structure

src/
  auth/
  users/        # CRM users (admins, promoters)
  players/      # Game players
  economy/      # currency system
  promotions/   # promo/referral tracking
  matchmaking/

## 🔐 Authentication Flow

1. Admin logs in → `/auth/login`
2. Backend validates credentials
3. JWT token returned
4. Frontend stores token
5. All protected routes require JWT

---

## 🔌 API Flow

Frontend (Next.js)
   ↓
NestJS API
   ↓
MySQL

---

## 🔄 Replacing PlayFab Features

| PlayFab Feature | Replacement |
|----------------|------------|
| Player Accounts | Player table |
| Statistics | PlayerStats |
| Virtual Currency | PlayerCurrency |
| UserData | PlayerData |
| Segments | SQL filters |
| Admin APIs | NestJS services |

## ⚠️ Security Rules

- Never expose DB directly
- Always hash passwords (bcrypt)
- Validate inputs (Zod / DTOs)
- Log all admin actions
